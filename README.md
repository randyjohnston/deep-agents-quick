# Deep Agents Quick Start

A research agent built on [Deep Agents](https://docs.langchain.com/oss/python/deepagents/) (LangGraph) that searches the web and writes polished reports. Supports Anthropic and Ollama providers via profile-based configuration.

## Setup

```bash
uv sync
cp .env.example .env  # then fill in your keys
uv run python agent.py
```

## Environment variables

Copy `.env.example` to `.env` and set the following:

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | For Anthropic | Your Anthropic API key |
| `TAVILY_API_KEY` | Always | Web search API key ([tavily.com](https://tavily.com)) |
| `MODEL` | No | Model string — see below |

## Switching model providers

The `MODEL` environment variable selects the provider and model. Two providers are pre-configured:

**Anthropic** (default):
```
MODEL=anthropic:claude-sonnet-4-6
```

**Ollama** (local):
```
MODEL=ollama:qwen3:27b
```

Ollama must be running locally (`ollama serve`) with the model pulled:
```bash
ollama pull qwen3:27b
```

To add a new provider, register a `ProviderProfile` in `app/profiles.py` and set `MODEL=<provider>:<model>`.

## Deploy to LangSmith

This repo includes a `langgraph.json` and `Dockerfile` ready for [LangSmith Deployments](https://docs.smith.langchain.com/self_hosting).

### Prerequisites

- A LangSmith account with Deployments access
- `LANGSMITH_API_KEY` (find it at **smith.langchain.com → Settings → API Keys**)
- `langgraph-cli` installed:

```bash
uv tool install "langgraph-cli[inmem]"
```

### Deploy

```bash
langgraph deploy --name deep-agents-quick
```

The CLI reads `langgraph.json`, builds the image, and pushes it to your LangSmith workspace. On first run it will prompt you to authenticate; set `LANGSMITH_API_KEY` in your environment to skip the prompt.

### Set secrets in the deployment

After deploying, add your API keys as environment variables in the LangSmith UI (**Deployments → your deployment → Settings → Environment variables**):

| Variable | Notes |
|---|---|
| `ANTHROPIC_API_KEY` | Required when using the Anthropic provider |
| `TAVILY_API_KEY` | Required — used by `internet_search` |
| `MODEL` | Optional — defaults to `anthropic:claude-sonnet-4-6` |

### Self-hosted deployments

For self-hosted LangSmith (or any environment where you bring your own container registry), build and push the image manually before deploying:

```bash
# Build the image
langgraph build -t myapp:latest

# Tag and push to your registry (example: GCP Artifact Registry)
docker tag myapp:latest us-docker.pkg.dev/<project>/<repo>/deep-agents-quick:latest
docker push us-docker.pkg.dev/<project>/<repo>/deep-agents-quick:latest
```

Then point your LangSmith deployment at the pushed image. The `Dockerfile` in this repo (generated by `langgraph build`) is also checked in so you can customise it or build it directly with `docker build`.

**Fixing `exec format error` on Apple Silicon (M-series Macs)**

If the deployed container fails with `exec /bin/sh: exec format error`, the image was built for the wrong CPU architecture. LangSmith servers expect `linux/amd64`. Fix it by enabling the multi-platform BuildKit builder before building:

```bash
docker buildx create --use
langgraph build --tag <registry>/<repo>/deep-agents-quick:0.1 --platform linux/amd64
docker push <registry>/<repo>/deep-agents-quick:0.1
```

### Call the deployed agent

```python
from langgraph_sdk import get_client

client = get_client(url="<your-deployment-url>", api_key="<your-langsmith-api-key>")
thread = await client.threads.create()
run = await client.runs.create(
    thread["thread_id"],
    "agent",
    input={"messages": [{"role": "user", "content": "Research the latest in agentic AI"}]},
)
```
